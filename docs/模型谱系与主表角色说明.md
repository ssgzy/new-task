# 模型谱系与主表角色说明

## 目的
- 这份文档用于把当前候选模型按“方法路线、基座、实验角色、当前状态”统一整理。
- 目标不是把所有模型都写成同一种方法，而是明确：
  - 哪些属于知识蒸馏路线
  - 哪些属于紧凑 base model 的直接对齐路线
  - 哪些只是边界候选、负例或参考项

## 核心说明
- 当前候选覆盖两条主流路线：
  - 知识蒸馏：`Orca-2-7B`
  - 紧凑 base model 的直接对齐：`Mistral-7B-Instruct-v0.3`、`Qwen2.5-7B-Instruct`、`Yi-1.5-6B-Chat`、`ChatGLM3-6B`
- `Zephyr-7B-beta` 保留为 `DPO-only` 负例分析。
- `Lion-7B` 保留为历史锚点与协议失败案例。
- `MiniLLM-Llama-7B` 保留为 `base_distilled baseline`，不并入当前统一 instruction 主表。

## 当前主表角色解释
- `formal main table`
  - 指当前已经通过 `screen200` 冻结门槛，可作为正式主表 validation883 候选继续推进。
- `boundary-pass`
  - 指接近门槛，但仍有一个关键指标差一点未过；报告里可以透明列出，但不应伪装成正式通过。
- `negative case`
  - 指早期看起来可用，但在更严格规模下暴露出协议层结构性不足，适合作为反例分析。
- `reference`
  - 指保留作方法谱系或历史连续性参考，不直接放进当前 formal main table。
