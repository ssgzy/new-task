# 准入筛选总结

## 目标
- 在 `val_screen200` 上对全部候选模型做 benchmark 准入筛选。
- 通过筛选的模型再进入完整 validation `883`。

## 准入门槛（暂定）
- `runtime_success >= 95%`
- `format_ok >= 70%`
- `valid_parse >= 70%`
- `truncation_without_answer_rate <= 5%`

## 正式记录指标
- `EM`
- `TM`
- `FormatOK`
- `ValidParse`
- `avg_latency_ms`
- `tok/s`
- `peak_vram`
- `mean_output_tokens`

## 当前状态
- 脚本已完成：`scripts/run_qualification_v1.py`
- 六个模型现均已可本地加载
- 主表正式筛选等待长度校准完成并冻结 `max_new_tokens`
- 目标产物：`outputs/qualification_summary.csv`

## 运行策略
- 本轮先对 `main` 组执行准入筛选。
- 脚本现支持：
  - `--groups main`
  - `--labels "<模型名>"`
  - `--resume`
  - `--provisional`
  - `--screen-only`

## provisional 规则
- 单模型或临时检查结果默认写到 `outputs/provisional/`
- canonical `outputs/qualification_summary.csv` 只保留正式主表聚合结果

## 当前执行约束
- 在当前这一步，只执行主表 `val_screen200`
- 为避免提前进入 `validation883`，应使用：
  - `conda run -n Lion python scripts/run_qualification_v1.py --groups main --max-new-tokens <冻结档位> --resume --screen-only`

## 备注
- Lion 可以作为历史锚点保留。
- 扩展表模型不参与首轮 student selection。
