# 组会汇报资料清单

## 用途说明
- 这份清单用于：
  - 给老师做组会汇报时直接展示
  - 作为 `NotebookLM` 的最小上传包
- 当前建议优先围绕这条叙事链：
  - 协议冻结
  - `screen200` 准入筛选
  - `validation883` 最终结果
  - 主表与失败案例定位

## 第一组：必须展示的核心文件
- `outputs/provisional/validation883_assigned/validation883_merged_results.csv`
  - 用途：最终主结果总表，最适合直接贴进 PPT 或导入表格软件
- `outputs/provisional/validation883_assigned/validation883_final_summary.md`
  - 用途：最终结果解释版，含主表/失败案例定位
- `docs/当前主表建议.md`
  - 用途：说明为什么主表最终保留 `Qwen2.5 / Mistral / Yi / Orca`
- `docs/模型谱系与主表角色说明.md`
  - 用途：解释模型路线差异，不把所有模型写成同一类
- `docs/评分逻辑与报告写作指南.md`
  - 用途：解释 `Strict TM / Relaxed TM / Gap` 等指标
- `docs/实验协议_v1.md`
  - 用途：说明 prompt、parser、decode、冻结口径
- `docs/准入筛选总结.md`
  - 用途：说明为什么先做 `screen200`，以及哪些模型通过/未通过

## 第二组：建议一并上传给 NotebookLM 的文件
- `outputs/provisional/screen200_6model_20260407/screen200_final_summary.md`
  - 用途：给 NotebookLM 补足“validation 之前为什么保留这批模型”的上下文
- `outputs/provisional/calibration_report.6model_mainstream_20260407.csv`
  - 用途：补足 `max_new_tokens=256` 的长度校准背景
- `outputs/provisional/validation883_assigned/Qwen2.5-7B-Instruct/summary.json`
- `outputs/provisional/validation883_assigned/Mistral-7B-Instruct-v0.3/summary.json`
- `outputs/provisional/validation883_assigned/Yi-1.5-6B-Chat/summary.json`
- `outputs/provisional/validation883_assigned/Orca-2-7B/summary.json`
- `outputs/provisional/validation883_assigned/ChatGLM3-6B/summary.json`
  - 用途：保留每个模型的原始 summary，便于 NotebookLM 读取细粒度指标

## 第三组：组会现场可按需展示的说明文档

- `docs/会话日志.md`

- 当前正式主表模型是：
  - Qwen2.5-7B-Instruct

  - Qwen2.5 > Mistral > Yi > Orca > ChatGLM3
