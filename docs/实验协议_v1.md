## Parser v1
- 统一只认最后答案行。
- 不为单个模型编写特殊 parser。
- 计划记录字段：
  - `format_ok`
  - `valid_parse`
  - `pred_value`
  - `pred_unit_type`
  - `answer_line_raw`
  - `multiple_answer_lines`
  - `truncated_suspect`
  - `parse_error_reason`
- `truncated_suspect` 冻结规则：
  - 文本末尾启发式命中
  - 或者 `new_tokens == max_new_tokens` 且缺失 `Answer:` 行

## Gold 口径
- `raw_answer` 保留原始答案字段值，优先取 `qa.answer`，缺失时回退到 `qa.exe_ans`，并在 `source_answer_field_name` 中记录来源。
- `gold_numeric` 统一标准化规则：
  - 去掉 `$`、`,`、`%`
  - 处理括号负号
  - `yes -> 1.0`
  - `no -> 0.0`
- `qa.answer` 与 `qa.exe_ans` 不可视为同义字段。validation `623/883` 条存在数值差异，主要来自百分比缩放或四舍五入差异，因此评测 gold 以 `raw_answer` 标准化结果为准。

## Context 构造
- 由 `scripts/finqa_protocol_v1.py` 固定实现。
- 拼接顺序：
  - `### Pre Text`
  - `### Table`
  - `### Post Text`
- `table` 逐行转为 `cell1 | cell2 | ...` 形式。
- `context_char_len` 以该完整 context 串的字符数计算。
